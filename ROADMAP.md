# Portal Roadmap

This roadmap reflects the current direction of the project, not a fixed promise.

## Near term

### Product polish

- Improve first-run onboarding
- Make pairing clearer for non-technical users
- Tighten the macOS control app layout and copy
- Add better connection-state feedback

### Windows setup flow

- Stabilize installer sharing from the Mac app
- Improve Windows companion install and update flow
- Reduce manual steps for first connection

### Input quality

- Improve cursor edge handoff behavior
- Refine return-edge behavior on macOS
- Expand keyboard mapping coverage
- Continue latency tuning

## Mid term

### Multi-display support

- Better Windows display detection
- Better Mac display restoration
- More predictable layout mapping between machines

### Transport and reliability

- Harden connection setup
- Improve reconnect behavior
- Add safer message validation
- Reduce assumptions about local network shape

### Sharing and workflow features

- Clipboard improvements
- Better status and diagnostics
- More complete installer and pairing UX

## Longer term

### Security

- Add stronger transport security
- Revisit trust and pairing model
- Reduce sensitive local-system assumptions

### Distribution

- Define the long-term split between direct distribution and any future store-friendly variant
- Improve release packaging for both macOS and Windows

### Broader platform story

- Evaluate whether Portal should remain a focused Windows-to-Mac tool or grow into a more general cross-platform input-sharing product
